# -
开源安全奖励计划参赛项目

一．项目简介

  本项目提出了一种针对关系数据库水印的暴力攻击防御算法，旨在解决现有水印技术在不可信环境下所面临的关键威胁：参数泄露导致的暴力破解成功率飙升。
  我们的方法确保只有当所有参数完全正确时，水印才能被精准恢复，而任何形式的参数猜测都会触发噪声注入，使攻击者无法获得有意义的水印信号，从而彻底阻断暴力攻击的可行性。
                
二．研究背景

  随着大数据时代的到来，数据已成为核心生产要素。数据库作为数据的承载、管理与价值挖掘的基石，其重要性愈发凸显。然而，数据库中的数据面临着盗用、篡改和非法复制等安全威胁，保障数据库数据安全性已刻不容缓。
  水印技术作为一种有效的数字版权保护手段，能够在数据库中嵌入特定版权信息，当数据库数据被盗用或破坏时，通过检测水印可以验证数据的所有权或完整性。
  暴力攻击是最常用的水印攻击方式之一。攻击者通过计算机辅助方法猜测用于水印同步的参数值，来进行水印的暴力攻击破解。在非可信环境下，比如通过互联网开放数据访问的数据库系统，水印同步参数泄露风险增大，暴力攻击的成功率会上升。一旦部分参数泄露，攻击者就可能大致恢复出水印图像。

三．项目具体内容
 
（1）提供参数脆弱性分层模型，并基于此构建防御机制
  我们根据参数在暴力攻击中的脆弱程度，将所有参数分为三类：
  1.PL1（关键参数）：密钥等极难猜测、泄露极危险的参数。
  2.PL2（主观参数）：猜错后会造成明显噪声，但某些图像结构可能保留。
  3.PL3（可复现参数）：猜中后可复原明显结构的参数（如水印尺寸、元组占比TF等）。
  我们算法主要针对PL3参数，其理由是：PL3参数最容易泄露或被猜测，而且猜中PL3参数通常能直接暴露大量水印结构。因此它们必须被严格验证。这种设计确保：攻击者无法跳过PL1与PL2，只依靠PL3参数猜测就恢复水印。

（2）识别并阻断攻击者通过“猜测参数”来同步水印的可能性
  在传统水印技术中，只要攻击者猜对部分参数，就可能成功推断出大致水印，从而检测、删除或伪造水印。我们引入验证信号机制，使得攻击者无法通过暴力攻击进行参数猜测从而提取水印。
  具体而言，验证信号包括PL3中的全部参数。我们的算法在水印嵌入前在数据库中先嵌入验证信号。验证信号和水印信号分区嵌入，互不干扰，使得嵌入区域更清晰，减少破坏性冲突。
  在水印检测阶段，先验证攻击者提供的参数是否完全真实。只要其中任一参数值错误，我们就立即触发惩罚机制，破坏水印同步。仅在全部参数均正确情况下，才可正确提取水印。这一机制确保部分参数泄露将不再足以让攻击者定位水印。
  当检测到存在参数不匹配的情况，惩罚机制对其做出实时响应，在每次检测水印位时，同时生成混沌序列噪声，并注入该位的检测结果中。此时，攻击者看到的水印图像将被彻底污染。噪声是类混沌生成，因此不可预测、不可逆。即使攻击者无限次尝试，也难以从错误参数中恢复有效水印，这极大增加了攻击者暴力攻击的成本。

（3）保障数据可用性，水印不可察觉性和提取准确性
  水印源为一张象征版权的二值图像。通过密钥，主键和其它相关参数选取部分元组的部分属性进行水印嵌入，避免对每个元组的大量修改，控制失真，减轻水印嵌入对数据库数据可用性的影响，同时也保障了其不可察觉性。
  提取水印时，使用投票机制得到最终水印，避免了正常对数据库的修改对水印造成的影响，保障了水印提取的准确性。
